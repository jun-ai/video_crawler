# 本地镜像打包与部署指南

## 一、打包镜像文件

### 后端镜像
```bash
cd D:/PythonTools/PythonProject/video_crawler/backend
docker build -t english-learning-backend:latest .
docker save english-learning-backend:latest | gzip > ../english-learning-backend.tar
```

### 前端镜像（已预构建 dist）
```bash
cd D:/PythonTools/PythonProject/video_crawler/frontend
docker build -t english-learning-frontend:latest .
docker save english-learning-frontend:latest | gzip > ../english-learning-frontend.tar
```

### 一键打包脚本
```bat
@echo off
cd /d D:\PythonTools\PythonProject\video_crawler

REM 后端
cd backend && docker build -t english-learning-backend:latest .
cd .. && docker save english-learning-backend:latest | gzip > english-learning-backend.tar

REM 前端
cd frontend && docker build -t english-learning-frontend:latest .
cd .. && docker save english-learning-frontend:latest | gzip > english-learning-frontend.tar

echo 打包完成！
pause
```

## 三、服务器部署步骤

### 1. 上传配置文件
需要上传以下文件到 `/opt/english-learning/`：

```
deploy/prod.env          →  .env
deploy/docker-compose.prod.yml  →  docker-compose.yml
deploy/nginx/nginx.conf   →  deploy/nginx/nginx.conf
```

### 2. SSH 到服务器执行
```bash
cd /opt/english-learning

# 停止旧服务
docker-compose down

# 加载后端镜像
docker load -i english-learning-backend.tar

# 加载前端镜像
docker load -i english-learning-frontend.tar

# 复制 .env
cp deploy/prod.env .env

# 启动服务
docker-compose up -d

# 检查状态
docker-compose ps
```

## 五、常见问题

### 1. 前端构建失败
- 原因：Docker 内 npm 构建缺少 native binding
- 解决：使用已构建好的 `dist` 目录，`frontend/Dockerfile` 已更新

### 2. 镜像加载失败
```bash
# 检查文件完整性
ls -lh english-learning-*.tar

# 重新加载
docker load -i english-learning-backend.tar
```

### 3. 服务启动失败
```bash
# 检查 .env 配置
cat .env

# 检查日志
docker-compose logs -f backend
```

### 4. HTTPS 不生效
```bash
# 检查证书文件
ls -la deploy/nginx/ssl/

# 确认证书存在
fullchain.pem
privkey.pem
```
