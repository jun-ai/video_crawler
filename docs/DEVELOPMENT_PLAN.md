# 口语学习网站优化开发计划

## 一、整体目标

将当前系统优化为参考设计风格，提升用户体验，增强学习效果。

## 四、设计规范

### 颜色系统
```css
/* 主题色 */
--primary-color: #4CAF50;        /* 绿色 - 高亮、按钮 */
--primary-light: rgba(76, 175, 80, 0.15);
--accent-color: #2196F3;         /* 蓝色 - 链接、图标 */

/* 文字颜色 */
--text-primary: #1a1a1a;         /* 英文 - 深灰/黑 */
--text-secondary: #666666;       /* 中文 - 浅灰 */
--text-tertiary: #909090;        /* 辅助文字 */

/* 背景颜色 */
--bg-primary: #ffffff;
--bg-secondary: #f9f9f9;
--bg-tertiary: #f1f1f1;
--bg-hover: rgba(0, 0, 0, 0.04);

/* 边框颜色 */
--border-color: #e5e5e5;
```

### 字体规范
```css
/* 英文字体 */
font-family: 'Inter', 'SF Pro Display', -apple-system, sans-serif;

/* 中文字体 */
font-family: 'PingFang SC', 'Microsoft YaHei', sans-serif;

/* 字号 */
--font-size-xs: 12px;    /* 时间戳、标签 */

--font-size-lg: 16px;    /* 英文字幕 */
```

## 六、开发顺序建议

```
Day 1: Phase 1 - 点读跳转功能
  ├── 上午：后端改动 + API测试
  └── 下午：前端跳转逻辑 + 动画

Day 2: Phase 2 - 布局重构
  ├── 上午：三栏布局HTML结构
  └── 下午：各区域样式 + 响应式

Day 3: Phase 3 - UI美化
  ├── 上午：字幕区细节优化
  └── 下午：词汇区优化 + 整体调优
```

---

## 七、注意事项

1. **保持现有功能**：所有改动不能影响现有功能
2. **渐进式改造**：先完成核心功能，再优化细节
3. **移动端优先**：响应式适配要充分考虑移动端体验
4. **性能优化**：避免过多的DOM操作和重绘
5. **代码复用**：提取公共组件和样式
